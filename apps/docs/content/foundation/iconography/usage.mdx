# Usage

## The `<Icon>` Component

The `<Icon>` component is the only way to render an icon in the KISA design
system. Never import a Lucide icon directly into a component.

```tsx
// Default size (md = 20px), decorative
<Icon name="arrow-right" />

// Explicit size
<Icon name="arrow-right" size="lg" />

// Semantic — announced by screen readers
<Icon name="x" label="Close" />
```

**Props:**

| Prop | Type | Default | Description |
|---|---|---|---|
| `name` | `string` | required | Lucide icon name in kebab-case |
| `size` | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'` | `'md'` | Size from the 5-step scale |
| `label` | `string` | — | Accessible label; omit for decorative icons |
| `className` | `string` | — | Layout utilities only (`block`, `flex-shrink-0`). Never use for color or sizing — those are controlled by parent color and the `size` prop. |

---

## Icons Are Never Interactive

The `<Icon>` component renders an SVG element. SVGs are not interactive — they
have no keyboard focus, no click semantics, and no accessible role that signals
interactivity to assistive technology.

Always wrap an interactive icon in a `<button>` or `<a>`. The wrapper provides
the interaction model, the keyboard target, and the accessible label.

```tsx
// ✅ correct — button provides interaction semantics
<button aria-label="Close">
  <Icon name="x" />
</button>

// ❌ wrong — icon has no interaction semantics
<Icon name="x" onClick={close} />
```

The wrapper also provides the touch target. Interactive elements must have a
minimum tap area of 44×44px. The `<Icon>` component itself is never 44px — the
button or link around it is.

## Icon + Text

When an icon appears next to a text label, use `flex items-center gap-2` as the
layout pattern. This aligns the icon optically with the text cap height and
provides consistent spacing.

```tsx
<button className="flex items-center gap-2">
  <Icon name="plus" size="sm" />
  <span>Add member</span>
</button>
```

The `gap-2` (8px) spacing works for most pairings. Use `gap-1` (4px) for
compact UI like tags or badges, and `gap-3` (12px) for larger display contexts.

---

## Icon-Only Interactive Elements

When an icon is the only content inside a button or link, provide an `aria-label`
on the wrapper so screen reader users know what the control does. Additionally,
wrap the button in a `<Tooltip>` for sighted users who may not recognize the
icon.

```tsx
<Tooltip content="Close dialog">
  <button aria-label="Close dialog">
    <Icon name="x" />
  </button>
</Tooltip>
```

The `aria-label` on the button and the tooltip content should be identical. Do
not provide a `label` prop on `<Icon>` when the button already has `aria-label`
— that creates redundant announcements.

## Color

Icons inherit `currentColor`. To change an icon's color, change the text color
of its container using semantic tokens.

```tsx
// Muted info icon
<span className="text-muted-foreground">
  <Icon name="info" />
</span>

// Error state
<span className="text-error">
  <Icon name="alert-triangle" />
</span>
```

Never pass a color prop to `<Icon>`. Never set `color` or `fill` inline on the
icon. If you find yourself wanting to pass a color directly, the color you want
already exists as a semantic token — reach for that instead.

---

## Disabled Icons

Disabled icons use `text-disabled-foreground` — the same token as disabled text. Never reduce size or weight to communicate disabled state. Only color changes.

```tsx
<span className="text-disabled-foreground">
  <Icon name="lock" />
</span>
```

## Responsive Sizing

Never apply breakpoint prefixes directly to icon size. Icon size is determined by component context — the component the icon lives in — not by the viewport width.

If a component changes size across breakpoints, the icon size change is encapsulated inside that component's variant logic. The `<Icon>` itself receives a fixed `size` prop.

```tsx
// ✅ correct — component variant controls icon size internally
<Button size="sm"> {/* Button uses sm icon internally on mobile */}
  <Icon name="plus" size="sm" />
</Button>

// ❌ wrong — breakpoint prefix on the icon directly
<Icon name="plus" size="sm" className="md:w-5 md:h-5" />
```

## Do's and Don'ts

### Wrapping interactive icons

<DoDont>
  <Do>
    <p>Wrap interactive icons in a <code>button</code> or <code>a</code>. The wrapper carries the interaction semantics and accessible label.</p>
    <pre><code>{`<button aria-label="Delete">
  <Icon name="trash-2" />
</button>`}</code></pre>
  </Do>
  <Dont>
    <p>Attach event handlers directly to <code>Icon</code>. The SVG element has no button role and is not keyboard-reachable by default.</p>
    <pre><code>{`<Icon name="trash-2" onClick={handleDelete} />`}</code></pre>
  </Dont>
</DoDont>

### Sizing icons

<DoDont>
  <Do>
    <p>Use the <code>size</code> prop from the defined scale. Every size maps to a predictable pixel value and 4px grid step.</p>
    <pre><code>{`<Icon name="search" size="sm" />`}</code></pre>
  </Do>
  <Dont>
    <p>Apply Tailwind text size classes to icons. Font-size utilities are not reliable for SVG sizing and produce unpredictable results.</p>
    <pre><code>{`<Icon name="search" className="text-xl" />`}</code></pre>
  </Dont>
</DoDont>

### Icon color

<DoDont>
  <Do>
    <p>Control icon color through the parent's text color using semantic tokens. The icon inherits <code>currentColor</code>.</p>
    <pre><code>{`<span className="text-error">
  <Icon name="alert-triangle" />
</span>`}</code></pre>
  </Do>
  <Dont>
    <p>Pass raw color values inline to an icon. This bypasses the semantic token system and cannot be updated globally.</p>
    <pre><code>{`<Icon name="alert-triangle" color="#ef4444" />`}</code></pre>
  </Dont>
</DoDont>
