# Layout

## Overview

Interfaces break when people make independent spacing decisions.
One developer reaches for `px-6`, another uses `px-[24px]`, a third
copies from a nearby component. Each choice is defensible in isolation.
Together, they create a page that feels assembled, not designed.

The KISA layout system is a small set of rules that removes those
decisions. Breakpoints, spacing, max-width, and gutters are defined
once. You apply them.

## Accessibility

The page shell must use semantic landmark elements so that screen reader users can navigate by region:

- `<header>` — site header (banner region: logo, site name, global utilities)
- `<nav>` — navigation landmarks (add `aria-label` if more than one `<nav>` exists on the page)
- `<main>` — the primary page content
- `<footer>` — site footer

Every page must include a **skip link** — a visually hidden anchor that becomes visible on focus and jumps to `<main>`. This allows keyboard and screen reader users to bypass repeated navigation on every page load.

```html
<a
  href="#main-content"
  class="sr-only focus-visible:not-sr-only focus-visible:fixed focus-visible:top-4 focus-visible:left-4 focus-visible:z-50 focus-visible:px-4 focus-visible:py-2 focus-visible:bg-surface focus-visible:text-text-primary focus-visible:rounded focus-visible:shadow-lg"
>
  Skip to main content
</a>
<main id="main-content">
  <!-- page content -->
</main>
```

These are implementation requirements for the page shell, not component-level concerns. They are configured once and apply globally.
